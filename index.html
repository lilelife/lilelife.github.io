java运行期优化

即时编译器（JIT）。将“热点代码”编译成平台相关的机器码，并进行分层优化。

热点代码：多次调用的方法；多次执行的循环体。检测方式（基于计数器；基于采用（查看栈顶））。

优化：

​ 方法内联：如果JVM监测到一些小方法被频繁的执行，它会把方法的调用替换成方法体本身

​ 冗余访问消除：

​ void foo(){

​ y=b.value;

​ z=b.value;

​ sum = y+z;

​ }

​ 消除后：

​ void foo(){

​ y=b.value;

​ z=y;

​ sum = y+z;

​ }

​ 复写传播：

​ 优化后：

​ void foo(){

​ y=b.value;

​ y=y;

​ sum = y+y;

​ }

​ 无用代码消除：

​ 优化后：

​ void foo(){

​ y=b.value;

​ sum = y+y;

​ }

公共字表达式消除：

​ int d=E*12 + a+（a+E）；

​ 优化后：

​ int d = E*13+a;

数组边界检测消除：

​ 数组每次进行访问时，foo[i] 都会对i进行检测，检测>0,<=lenth。

​ 编译时通过根据数据流分析i有没有越界。执行时不进行判断。

逃逸分析：

​ 相当于类型继承关系分析。分析对象动态作用域。

​ 当一个对象在方法内定义，肯定在外部方法中引用，例如作为调用参数传递到其他方法中，成为方法逃逸。如果真没一个对象不会逃逸到其他线程或方法，也就是别的方法或者线程无法通过任何途径进行访问该变量。不会逃逸时，可以进行如下优化：

​ 栈上分配，将对象内存在栈上分配，该对象可以随着方法的结束进行销毁，省略堆上面的gc。

​ 同步消除，线程同步比较耗时，当对象不会逃逸出线程时，说明不会被其他线程访问到，可以 省略变量的同步措施。

​ 标量替换，标量（int、long、等）无法再解析为更小的数据来表示。但对象不是一个标量。可以拆散 ，如果不会逃逸时，则可以使用若干个被这个方法使用的成员变量来替换。

java类文件结构

字节码是够成平台无关性的基石，java虚拟机不和包括java在内的任何语言绑定，只和“class 文件”这种特定的二进制文件格式关联。Class文件包含了java虚拟机指令集，和符号表以及其他若干辅助信息。

Class类文件的结构

任何一个Class文件都对应唯一一个类或者接口，但是类和接口并不一定定义在文件中。

Class文件是一组以8位字节为基础单位的二进制流，各个数据项紧促排列。根基java虚拟机规范，Class文件格式采用类似C结构的伪结构，该伪结构包括两种数据类型：无符号数，表。无符号数是基本的数据类型，以U1，u2，u4，u8来代表1字节，2字节，4字节等的无符号数，无符号数用于描述数字，索引引用，数量值或者字符串值。表使用多个无符号数或者其他表构成的复合数据类型，习惯使用“_info”结尾。

每个Class文件的头4个字节成为魔数（Magic number）,唯一作用是确定该文件是否为一个可以被虚拟机接受的Class文件。魔数后面的4个字节存储Class文件的版本号。包含次版本号与主版本号。在主版本号之后是常量池入口，常量池入口有一个u2类型的数据表示常量池计数值。常量池存放字面量和符号引用。字面量包含文本字符串，声明为final的常量值，符号引用包含类和接口的权限定名，字段的名称和描述符，方法的名称和描述符。Class文件中方法、字段都需要使用Constant_Utf8_info描述。常量池结束后，是两个字节的访问标识，标识是类还是接口，public ，abstract final等。接着包含，类索引，父类索引，字段表集合，方法表集合，属性表集合。

字节码指令简介

java虚拟机的指令由一个字节长度的、代表着某种特定操作含义的数字，也成操作码。以及多个代表此操作所需参数（操作数）构成。

​ 加载和存储指令。

​ 运算指令。

​ 类型转换指令。

​ 对象创建和访问指令。

​ 操作数栈管理指令。

​ 控制转移指令。 方法调用返回，异常处理，同步指令。

类加载机制

虚拟机类加载机制

整个类的加载过程分为：加载、验证、准备、解析、初始化、使用、卸载。其中验证、准备、解析统称为连接。

加载：

​ 类加载。非数组类，用户可以给自定义一个类加载器去控制字节流的获取方法（重写类加载器的loadClass（）方法）。对于数则类，不通过类加载器，是由java虚拟机直接创建的。

验证：

​ 确保Class文件字节流包含的信息符合虚拟机要求，并不会危害虚拟机自身的安全。

​ 文件格式验证，确保字节流可以解析到方法区；

​ 元数据验证，对字节码描述信息的验证，是否继承，父类，抽象类等；

​ 字节码验证，通过数据流和控制流分析，确定程序语义。对类的方法验证。

准备：

​ 正式为类变量分配内存并设置类变量初始值。内存分配在方法区。只分配 类变量（static修饰）,不包含实例变量。实例变量在对象实例时分别在堆中和对象一起。

​ publilc static int value = 123； //初始值为0.不是123 ,赋值为123是在putstatic指令编译时，存放在类构造器()方法中，在初始化阶段进行

解析：

​ 将常量池中的符号引用解析替换成直接引用。

​ 符号引用是以一组符号来描述引用的目标。

​ 直接引用是直接指向目标的指针，相对偏移量或间接定义到目标的句柄。

类和接口的解析；字段解析。

初始化：在进行初始化时，加载、验证、准备需要执行。情景：1 、遇到new getstatic putstatic invokestatic时，即new对象，读取设置静态字段，调用类的静态方法。

​ 2、使用lang.reflect包的反射调用时，类没有初始化，则需要。

​ 3、初始化一个类，其父类没有初始化，则初始化父类。接口不用

​ 4、虚拟机启动时，用户指定一个要执行的主类（包含main()方法的类），虚拟机会先初始化该类。

​ 5、使用jdk1.7的动态语言支持时，使用MethodHandle实例最后的解析结果REF_getstatic 、REF_putStatic的方法句柄时，该句柄所对应的类没有进行初始化时，虚初始化。

上面为主动应用。除此之外，所有引用类的方法都不会触发初始化，称为被动引用。通过子类引用父类的静态字段，不会导致子类初始化。常量在编译阶段会存入调用类的常量池，在引用时，没用直接引用到定位常量的类，直接从常量池中引用常量。

初始化时指向()方法。保证父类的该方法先执行，说明了父类的静态语句在子类前面执行。()方法不是必须执行的，当一个类没有静态语句块时，就没有对变量的赋值操作，编译器可以不Wie这个类生产()方法。接口不可以使用静态语句，但可以有变量初始化的赋值操作，接口不需要先执行父类的（）方法。虚拟机对一个类得()方法在对线程下正确枷锁与同步。保证多个线程初始化时，只能有一个线程去执行给类的()方法。

类加载器

​ 执行类加载过程的代码模块。

​ 双亲委派模型：类加载收到类加载请求，不会自己先加载，会将请求委派到父类加载器完成，每层都是，3层。

​ 1、启动类加载器，

​ 2、扩展类加载器

​ 3、应用程序类加载器（系统类加载器），负责加载用户类路径Classpath上面的指导类库。

java内存回收

Gc

1 对象是否存活

​ 1.1 引用计数算法：

​ 对象创建时，添加引用就计数器，每当引用时，计数器加一；引用失效是，计数器减一；当计数器的值为0时，不再使用。 缺点，难以解决对象之间的循环引用。

​ 1.2可达性分析算法：

​ 使用“GC roots”的对象作为起始点，以该节点，向下搜索，形成引用链，gc回收不再引用链上面的对象。

​ gc roots的对象，包括

​ 虚拟机栈（栈帧里的本地变量表）中引用的对象；

​ 方法区类静态属性应用的对象

​ 方法区常量应用的对象

​ 本地方法栈中JNI（Native方法）引用的对象

​ 对象在进行可达性分析后，发现没有与gc roots进行连接，则第一次标记，并进行筛选，判断是否需要执行finalize()方法，当对象没有覆盖该finalize方法时，或者finalize已经被系统调用了，则视为“没必要执行”，finalize方法时对象逃脱gc的唯一机会，且只能使用一次，。任何对象的finalize()方法只能被系统执行一次，可以通过override 该方法，在

protected void finalize() throws Throwable {

​ …

}

中进行对象自救，例如重新连接引用等。

2引用

​ jdk1.2后面，java对引用进行扩展。引用分为，

强引用（Strong Reference） 软引用（Soft Reference） 弱引用（Weak Reference） 虚引用（Phantom Reference）

​ 强引用，类似与 =new 的对象，强引用时，对象不被gc

​ 软引用，当系统内存溢出异常时，将这些引用对象放入第二次回收列表。如果这次回收时，还没有足够内存，抛出内存溢出异常。

​ 弱引用，无论内存是否足够，都进行回收

​ 虚应用，呵呵 你懂得 仅仅为了系统得到对象回收时的一个通知。。。。

3 方法区的回收

​ 方法区的回收包括 废弃常量和 无用的类

​ 无用的类需要满足三个条件：该类所有实例被回收，堆中不存在该类的对象；加载该类的ClassLoader已经被回收；该类的java.lang.Class对象没有在任何地方医用，无法通过反射访问该类的方法。

4垃圾收集算法

​ 4.1标记-清除算法

​ 标记，全部回收。缺点：两个步骤的效率都不高，标记清除后容易形成垃圾碎片

​ 4.2 复制算法

​ 内存被分为一个Eden空间，两个Survivor空间，比例8:1:1，使用时使用一个Eden和一个Surviror，当内存不够时，将两个使用的空间中的存活的对象复制到另一个Survivor中，然后将Eden区域的和先前使用的Sruvivor区域进行垃圾回收。

​ 应用是死亡对象多的，适用于新生代。

​ 4.3标记-整理算法

​ 使用于老年代， 标记阶段与“标记-清除”算法一致，让所有存活的对象都想一端移动，然后直接清除端边界以外的内存。

​ 4.4分代收集算法

​ 将堆根据对象存活周期，将内存分为新生代，老年代，根据不同的区域进行不同的收集算法。

java内存

.运行时数据区域

总体结构图：

​ 1.1 程序计数器 ，线程私有，分配给每个线程

​ 1.2java虚拟机栈： 线程私有，每个方法执行时，创建线程，与线程同步，存储局部变量表，操作数栈，动态链接，方法出口

​ 局部变量表存储基本数据类型，对象引用（reference类型）,returnAddress(指向一条字节码指令的地址),

​ 1.3本地方法栈：为虚拟机使用的native方法服务。

​ 1,4java堆：存放对象实体，数组的内存。“GC”堆 可以分为新生代，老年代。 线程共有。

​ 1.5方法区 ：线程共有， 各个线程共享的内存区域。存储类信息，常量，静态变量，即时编译后的代码。

​ 1.6运行时常量池：属于方法区，常量池通过类加载后，存储到运行时常量池（方法区）.存储字面量，符号引用。

.对象的创建

new时，常量池检测类的符号引用，没有进行类加载

，加载后，进行虚拟机为新生对象分配内存，内存大小加载后确定。内存分配使用“指针碰撞”，将指针向空闲区域挪动与对象大小相同的距离。另一种是“空闲列表”，内存空闲与已经使用的交错分布。

对象的内存分布

​ 对象头（运行时数据，类型指针：指向类的类型）；

​ 实例数据（存储字段信息）；

​ 对齐填充（对象大小规则为8字节整数倍）

对象的访问定位

​ 句柄访问：引用-》句柄池-》地址 。句柄包含对象实例数据，类型数据的具体地址信息。

​ 直接指针访问：引用-》地址。

​ 区别，句柄方式，reference中存储的是稳定句柄地址，对象移动实，不需要更改应用。直接引用的访问速度较快。
